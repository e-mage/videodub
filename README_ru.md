# Проект VideoDub - Аудио дубляж видео/подкастов

## Описание

Проект **VideoDub** предназначен для автоматического дубляжа аудио дорожки на другой язык.

Основная идея проекта - использовать возможности модели [Qwen3-TTS/Base](https://github.com/QwenLM/Qwen3-TTS#voice-clone) для озвучивания фраз на другом языке с сохранением тембра голоса и интонации говорящего. Модель поддерживает следующие языки: Chinese, English, Japanese, Korean, German, French, Russian, Portuguese, Spanish, Italian.

В проекте используются также следующие локально установленные библиотеки/модели/утилиты:

- Whisper (транскрибация),
- SBERT (разбивка на смысловые сцены),
- Ollama/TranslateGemma (перевод текста сцены на другой язык),
- FFMpeg (конвертация, склейка и извлечение аудио).

Процедура дубляжа в данном проекте состоит из следующих этапов:

- Главный скрипт для запуска проекта `video-dub.py` - производит первоначальную детекцию смысловых сцен фильма на основе предоставленных субтитров (или результата автоматической транскрибации аудио в субтитры (с помощью библиотеки `openai-whisper`)). Детектор смысловых сцен использует библиотеку и модели **SBERT embeddings**, которая позволяет искать близкие по смыслу предложения. Детектор НЕ использует видеоряд, поэтому вместо видео файла можно передать просто аудио дорожку (также через аргумент `--video` для скрипта `video-dub.py`).

- Транскрибация сцены, перевод сцены на другой язык и передача на озвучивание каждой фразы в сцене. Если при запуске скрипта `video-dub.py` не указан флаг `--no-cut`, то скрипт каждую найденную аудио-сцену вырезает и отправляет другому скрипту `scene-dub.py` на транскрибацию, перевод и озвучивание (TTS). Результаты озвучивания всех сцен скрипт `video-dub.py` собирает в конце в одну финальную аудио дорожку.

- Для озвучивания (Text-to-Speech) всех фраз в сцене, скрипт `scene-dub.py` изпользует вспомогательный скрипт `phrase-tts.py`.

В итоге проект **VideoDub** состоит из трёх Python-скриптов:

- `video-dub.py` (Главный скрипт / Детектор смысловых сцен)
- `scene-dub.py` (Транскрибирует и переводит сцену. При желании можно озвучить её с помощью скрипта 'phrase-tts.py'.)
- `phrase-tts.py` (дубляж TTS для списка фраз)

## Установка

### 1. Клонируйте проект из репозитория github

```bash
git clone https://github.com/e-mage/videodub.git
cd videodub
```

### 2. Создайте виртуальное окружение Python

```bash
python3.12 -m venv .venv
source .venv/bin/activate
```

### 3. Установите Qwen3-TTS

Подробнее прочитайте на домашней странице проекта: <https://github.com/QwenLM/Qwen3-TTS>

Установка `qwen3-tts`:

```bash
pip install -U qwen-tts
MAX_JOBS=4 pip install -U flash-attn --no-build-isolation
```

>**Внимание!!!** Установка библиотеки `flash-attn` занимает много времени и потребляет много оперативной памяти и ресурсов процессора. Поэтому ограничьте потребление ресурсов вашего компьютера, установив переменную среды `MAX_JOBS=4`.

#### Проверьте работу Qwen3-TTS в браузере

Запустите веб-сервис:

```bash
qwen-tts-demo Qwen/Qwen3-TTS-12Hz-1.7B-Base --ip 0.0.0.0 --port 8000
```

И затем откройте ссылку в браузере <http://localhost:8000>.

### Установите OpenAI-Whisper

```bash
pip install openai-whisper
```

### Установите sentence_transformers (aka SBERT)

Библиотека SBERT используется для поиска сходства между предложениями. Подробнее прочитайте:

<https://www.sbert.net/>

<https://huggingface.co/sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2>

Для установки:

```bash
pip install -U sentence_transformers
```

### Установите остальные зависимости Python

```bash
pip install -U librosa soundfile requests pyrubberband num2words pysrt numpy scikit-learn sentencesplit regex
```

### Установите ffmpeg

В вашей операционной системе должен быть установлен пакет `ffmpeg`. Для установки:

```bash
sudo apt install ffmpeg
```

### Установите ollama и скачайте модель translategemma:12b

<https://ollama.com/>

<https://ollama.com/library/translategemma>

```bash
curl -fsSL https://ollama.com/install.sh | sh
ollama pull translategemma:12b
```

## Требуемые ресурсы, скорость выполнения дубляжа

Проект тестировался на компьютере с 16 Гб ОЗУ и NVidia картой с 16 Гб VRAM. С такими параметрами время выполнения полного дубляжа обычно в 2 раза меньше длительности обрабатываемой аудио дорожки.

Без поддержки графической карты процедура дубляжа будет в разы дольше.

## Использование

### TL;DR

Для запуска полной процедуры дубляжа:

```bash
python video-dub.py --video my-chinese-video.mp4 --output out-russian --to-language Russian --max-scene-duration 60
```

Для создания только субтитров на языке оригинала (`transcribed.srt`) и списка смысловых сцен (`scenes_list.txt`) используйте флаг `--no-cut`:

```bash
python video-dub.py --video my-chinese-video.mp4 --output out-russian --to-language Russian --max-scene-duration 60 --no-cut
```

> **Обратите внимание!** Вы можете в любой момент прервать выполнение скрипта в терминале (Ctrl-C). Повторный запуск скрипта `video-dub.py` продолжит работу с прерванного места.

Иногда одна и несколько сцен могут не обработаться корректно, о чем скрипт в конце своей работы скажет. Повторный запуск скрипта часто решает такую проблему.

### Результаты

В папке, указанной параметром `--output`, будут созданы следующией файлы:

- `transcribed.srt` - Полные транскрибированные субтитры на языке оригинала (фомат SRT). Если при запуске был передан параметр `--translate`, то субтитры будут на английском языке. При повторном запуске скрипта `video-dub.py` файл с субтитрами будет переиспользован, то есть повторная транскрибация производиться не будет.
- `scenes_list.txt` - Список смысловых сцен с указанием времени начала и конца, а также первой фразы в каждой сцене. При повторном запуске скрипта `video-dub.py` файл со списком смысловых сцен будет переиспользован, то есть повторный поиск сцен производиться не будет.
- `source_audio.wav` - Извлеченная стерео аудио дорожка из видео файла. Также может быть переиспользована при повторных запусках.
- `central_channel.wav` - Извлеченный центральный канал для случая когда в видео файле присутствует многоканальная аудио дорожка (AC3 - 5.1, 7.1). Центральный канал обычно содержит все разговоры в фильмах, поэтому только он будет использоваться для дубляжа. Также может быть переиспользован.
- `merged_audio.wav` - Аудио моно дорожка, содержащая дубляж, то есть все фразы, озвученные с помощью TTS, склееные вместе.
- `merged_subtitles.srt` - Файл с переведенными субтитрами в формате SRT.
- `final_mixed_audio.wav` - Финальная стерео дорожка, содержащая полный дубляж. То есть это смикшированные вместе оригинальная аудио дорожка (`source_audio.wav` или `central_channel.wav`) и дорожка с дубляжом (`merged_audio.wav`). Громкость оригинальной аудио дорожки уменьшена на -12dB.

> В папке с результатами могут остаться другие временные файлы из последней сцены.

Пока на данный момент скрипт `video-dub.py` автоматическии НЕ добавляет созданную дорожку с дубляжом и переведенные субтитры в оригинальный видео файл. Но это можно сделать с помощью утилиты `ffmpeg`, например, для случая дубляжа с китайского языка на русский:

```bash
cd ./out-russian
ffmpeg -i final_mixed_audio.wav final_mixed_audio.aac
ffmpeg -i ../my-chinese-video.mp4 -i final_mixed_audio.aac -i transcribed.srt -i merged_subtitles.srt -map 0:v -map 0:a -map 1:a -map 2:s -map 3:s -c copy -metadata:s:a:0 language=zho -metadata:s:a:1 language=rus -metadata:s:s:0 language=zho -metadata:s:s:1 language=rus my-chinese-video-rusdub.mkv
```

## ЧаВо

> Для чего производится разбивка на смысловые сцены? Почему нельзя целиком транскрибировать, перевести и озвучить?

- Главное ограничение здесь - перевод текста на другой язык. Обычно модели перевода, такие как translategemma, имеют очень небольшое контекстное окно, в которое не помещается целиком весь текст видео/подкаста. Перевод же отдельных фраз приводит к полной потере контекста для переводчика.

> Допустимо ли вообще клонировать тембр и интонации на другой язык?

- Да, допустимо. Об этом пишут сами разработчики модели `qwen3-tts`. Правда, не во всех случаях это работает идеально. Иногда в клонированный вариант пробиваются просодии из оригинального языка, и это звучит так, как будто бы человек хорошо говорит на неродном языке, но не может избавиться от родного акцента.

> Почему иногда тембр говорящего "плавает" от фразы к фразе, как будто говорят разные люди?

- К сожалению, невозможно зафиксировать тембр при клонировании разных фраз в работе модели `qwen3-tts/Base`. Модель хороша, но еще не совершенна.

> Почему иногда выпадают/отсутствуют некоторые фразы в оригинальных субтитрах, или переведенных субтитрах, или в аудио дорожке дубляжа?

- Отсутствие фразы в субтитрах скорее всего связано с тихой, неразборчивой или зашумленной речью в оригинальной аудио дорожке. А вот если фраза присутсвует в субтитрах, но её нет в дубляже, то это означает, что модель TTS не справилась с озвучкой. Посмотрите следующий вопрос.

> Если не понравился дубляж в какой-то одной сцене, можно ли передублировать только эту сцену?

- Да, можно. Для этого надо удалить в папке с результатами все файлы, связаные с конкретной сценой с номером NNN - `scene_NNN*.*`, и запустить скрипт повторно. Обратите внимание, что каждый раз дубляж будет работать немного по-разному - и транскрибация, и перевод, и озвучка. Для того, чтобы узнать номер сцены, загляните в файл `scenes_list.txt`.
